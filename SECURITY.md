# Security Policy

## Reporting a Vulnerability
Please report security issues privately.

    1. Do **not** open a public issue for sensitive vulnerabilities.
    2. Include reproduction steps, affected components, and potential impact.

## Supported Versions
Until a stable release is published, treat `main` as active development.

## Security Features (planned/expected)
    1. Tool allowlists and policy enforcement
    2. Redaction for sensitive data in logs
    3. Sandboxed tool execution wrappers
    4. Audit trail for high-risk actions
