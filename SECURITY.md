# Security Policy

## Reporting a Vulnerability
Please report security issues privately.

- Do **not** open a public issue for sensitive vulnerabilities.
- Include reproduction steps, affected components, and potential impact.

## Supported Versions
Until a stable release is published, treat `main` as active development.

## Security Features (planned/expected)
- Tool allowlists and policy enforcement
- Redaction for sensitive data in logs
- Sandboxed tool execution wrappers
- Audit trail for high-risk actions
